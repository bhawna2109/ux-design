Writing blog posts

I love articulating the 'What', 'Why', and 'How' for the tools I design. At Postman, I got the opportunity to write stories on the Postman blog for features that I worked on and shipped as the Design Lead (sole designer).


Librarian: building a serverless Slack app using Postman and Airtable

As someone who was responsible to design Postman, it was my top responsibility to learn about and internalize the product as much as possible. I really wanted to pick up a real use case and actually build something using Postman.

One day I got this idea of making a serverless library manager Slack app using Postman and Airtable, and immediately started building it with the support of my peers. After a few days – and a LOT of debugging – I felt elated to see the Slack command actually working! This story has also been published on the airtable blog.



Data Editor

Redesigned the data manipulation component – a core component in Postman. I loved working on this project and believe that it showcases my best work.



Descriptions for request attributes

Built on top of the newly designed Data Editor – ability to add descriptions for request attributes.



Mock responses in Postman by using Examples

Examples was a new abstraction in Postman – this project involved dealing with ambiguity and defining a lot of unknowns by asking the right questions. Additionally, we had to deprecate an older feature and transition to this new one. This was particularly challenging for me because changing something which millions of people are already using requires careful design decisions.



Autocomplete and tooltips for variables are here

Making workflows efficient is at the core of Postman's UX principles. Supporting tootips and autocomplete for variables is something that is present in all major development environments and it was long awaited in Postman.
